# Соглашения по именованию

## Наименование переменных
Наименование переменных по принципу camelCase.

Примеры:
- `count` - Наименование переменной из одного слова 
- `isDataLoaded` - Наименование переменной из двух и более слов

## Наименование файлов

Примеры:
- `game` - Наименование файла из одного слова 
- `server_test` - Наименование файла из двух и более слов

## Наименование веток при разработке:

`<номер-задачи>-<тип>/<краткое-описание>`

Примеры:
- `00-docs/` - Изменения в документации.
- `01-chore/` - Технические задачи, не связанные с бизнес-логикой (обновление зависимостей, настройка инструментов).
- `02-feature/` - Новая функциональность. Основной тип для разработки.
- `03-bugfix/` - Исправление бага.
- `04-refactor/` - Рефакторинг кода без изменения функциональности.
- `05-task/` - Универсальный тип для любых задач, которые не подходят под другие категории.
